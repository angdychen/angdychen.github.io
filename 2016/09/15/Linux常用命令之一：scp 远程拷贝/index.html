Linux常用命令之一 ：scp 跨机远程拷贝

Linux
  • Linux常用命令

scp是secure copy的简写，用于远程文件拷贝(cp复制命令和其类似，不过cp命令只是在本机内进行拷贝，不能跨机器进行文件传输)，而且scp传输是加密的，常用于本地与服务器间文件的传递。

命令格式：scp [参数] [原路径] [目标路径]

命令格式：

命令格式：
        scp [参数] [原路径] [目标路径]
命令参数:
        -1 强制scp命令使用协议ssh1
        -2 强制scp命令使用协议ssh2
        -4 强制scp命令只使用IPv4寻址
        -6 强制scp命令只使用IPv6寻址
        -B 使用批处理模式（传输过程中不询问传输口令或短语）
        -C 允许压缩。（将-C标志传递给ssh，从而打开压缩功能）
        -p 留原文件的修改时间，访问时间和访问权限。
        -q 不显示传输进度条。
        -r 递归复制整个目录。
        -v 详细方式显示输出。scp和ssh(1)会显示出整个过程的调试信息。
        这些信息用于调试连接，验证和配置问题。
        -c cipher 以cipher将数据传输进行加密，这个选项将直接传递
        ssh。
        -F ssh_config 指定一个替代的ssh配置文件，此参数直接传递
        ssh。
        -i identity_file 从指定文件中读取传输时使用的密钥文件，
        此参数直接传递给ssh。
        -l limit 限定用户所能使用的带宽，以Kbit/s为单位。
        -o ssh_option 如果习惯于使用ssh_config(5)中的参数传递方式，
        -P port 注意是大写的P, port是指定数据传输用到的端口号
        -S program 指定加密传输时所使用的程序。此程序必须能够理解
        ssh(1)的    选项。

使用范例

1.从本地将文件拷贝到服务器中

scp local_file remote_username@remote_ip:/remote_floder

scp /tmp/demo.txt root@123.123.123.123:/usr/
说明：复制本地/tmp/目录下的文件demo.txt到远程服务器123.123.123.123的/usr/目录下

2.从本地将整个文件夹拷贝到服务器中

scp -r local_floder remote_username@remote_ip:/remote_floder

scp -r /tmp/demo/ root@123.123.123.123:/usr/
说明：复制本地/tmp/目录下的整个demo文件夹录下到远程服务器123.123.123.123的/usr/目录下

3.从远程将文件夹拷贝到本机

scp remote_username@remote_ip:/remote_file /tmp/

scp root@123.123.123.123:/usr/demo.txt /tmp/
说明：将远程远程服务器123.123.123.123中/usr/目录下的demo.txt文件复制到本地/usr/目录下

4.从远程将整个文件夹拷贝到本机中

scp -r remote_username@remote_ip:/remote_floder /tmp/demo/

scp -r root@123.123.123.123:/usr /tmp/demo/
说明：将远程服务器123.123.123.123的/usr/整个目录全部复制到tmp/demo目录下
